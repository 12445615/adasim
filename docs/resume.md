# Resume Version

## Project Entry

2025.02 - 2025.03  
自动驾驶可视化与嵌入式上位机平台 ADASim  
负责人  
技术栈：C++、Qt、Embedded Linux、Python、TCP/UDP、SocketCAN、SQLite、多线程、车辆运动学、BFS 聚类

## Resume Bullets

- 设计并实现面向 Embedded Linux 的 Qt/C++ 车载上位机，集成仿真画布、LiDAR 点云窗口、车辆状态面板、通信日志和回放滑条，实现车辆状态、传感器数据和算法输出的实时可视化。

- 基于 QTcpSocket 实现与 Python 算法节点的异步通信，设计一行一个 JSON 的控制报文协议，解决 TCP 流式传输中的粘包和半包问题，实现速度、转角等控制量对车辆运动学模型的实时驱动。

- 构建自行车运动学闭环仿真模型，根据外部控制节点输出的 speed/steering 实时更新车辆 x、y、yaw、里程和历史轨迹，并通过平滑视角跟随提升长距离仿真下的可视化稳定性。

- 基于 QUdpSocket 实现模拟 LiDAR 传感器接收模块，接收外部 UDP 点云数据并执行 BFS 欧式聚类，完成点云显示、障碍物簇识别和聚类数量统计，模拟真实雷达以太网数据接入链路。

- 设计交互式虚拟障碍物注入模块，支持运行时通过鼠标右键将屏幕坐标逆向转换为车辆局部点云，用于模拟突发障碍物和雷达误检场景；当前方安全区域存在障碍物时触发自动制动。

- 基于 SocketCAN 实现控制指令下发模块，将车辆速度、转角与自动制动状态封装为 CAN 报文，并在 Linux vcan 环境下完成通信链路验证，为后续 RK3588 对接真实底盘控制器提供接口基础。

- 基于 SQLite 设计本地运行记录系统，周期性持久化车辆状态、控制报文、点云统计和故障注入事件，支持对自动制动、虚拟障碍物注入和系统运行状态进行离线查询与复盘。

- 完成项目在 x86_64 Ubuntu 虚拟机下的编译与运行验证，形成面向 RK3588 ARM Linux 平台的部署方案，覆盖 Qt 运行环境、SocketCAN、systemd 自启动和全屏显示配置。

## Interview Summary

这个项目的核心不是单纯做自动驾驶算法，而是模拟真实车载嵌入式上位机的数据流：

```text
算法控制输入 TCP
传感器输入 UDP
控制输出 SocketCAN
本地记录 SQLite
Qt/C++ 实时可视化
```

可以重点讲：

- 为什么 TCP 控制协议要做行缓冲
- 为什么 UDP 更适合模拟高频传感器数据
- SocketCAN 如何对应真实底盘通信
- 虚拟障碍物如何模拟突发障碍和雷达误检
- SQLite 如何支持故障复盘
- Linux 虚拟机验证和 RK3588 部署的区别
